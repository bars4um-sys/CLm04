# Назначение

Одностраничное HTML-приложение, запускаемое двойным кликом по файлу в браузере.
Сравнивает две таблицы Excel (.xlsx/.xls/.xlsm/.xlsb) **построчно** и находит расхождения.

Два режима:
1. **Поиск дублей** в Авансовом отчёте по паре (Дата + Сумма) — только просмотр и экспорт.
2. **Сравнение** Авансового отчёта с Банком по столбцу «Сумма» — выводит строки Банка,
   для которых сумма не найдена в Авансовом отчёте.

Результат — две независимые таблицы на экране + кнопка скачивания каждой в Excel.

# Привязка к столбцам (фиксированные индексы)

Это **самое важное** место — при любых изменениях файлов Excel проверять именно его.
Индексы считаются с 0 (визуально «N-й столбец» → индекс N-1).

| Константа (в коде)       | Значение | Назначение                                    |
|--------------------------|----------|-----------------------------------------------|
| `DATE_COLUMN_AO`         | 1        | Дата в Авансовом отчёте (2-й столбец)         |
| `SUM_COLUMN_AO`          | 4        | Сумма в Авансовом отчёте (5-й столбец)        |
| `SUM_COLUMN_BANK`        | 1        | Сумма в Банке (2-й столбец)                   |

Ключ дублей — пара `(DATE_COLUMN_AO, SUM_COLUMN_AO)`.
Ключ сравнения с Банком — `SUM_COLUMN_AO` ↔ `SUM_COLUMN_BANK`.

Если пользователь скажет «дата в N-м столбце» — менять только эти константы.

# Поддержка двухуровневой шапки

`detectHeaderRow(rows)` — если первая строка полностью пустая, считается,
что это объединённая ячейка верхнего уровня группы (визуальная шапка
«Документ, подтверждающий производственные расходы»). Реальные подзаголовки
берутся из 2-й строки, данные — с 3-й. Иначе — шапка в 1-й строке, данные со 2-й.

# Нормализация значений

`normalizeSum(value)` — для сумм:
- убирает пробелы, заменяет запятую на точку, выкидывает всё кроме цифр/точки/минуса,
- парсит в `Number`, округляет до 2 знаков (до копеек).
- Возвращает `null` для пустых/нечисловых значений.
- Строки с `null` в ключевом столбце **пропускаются** при поиске дублей.

`normalizeDate(value)` — для дат:
- Поддерживает `Date`-объект, ISO `YYYY-MM-DD`, `DD.MM.YYYY` и `DD/MM/YYYY`,
  Excel-серийный номер дней (1..100000).
- Возвращает строку в формате `YYYY-MM-DD` или `null`.

# Логика шага 1 — дубли

1. Группируем строки АО по ключу `normalizeDate(row[DATE_COLUMN_AO]) + '|' + normalizeSum(row[SUM_COLUMN_AO])`.
2. Оставляем только группы с `rows.length > 1`.
3. Внутри группы сортируем строки по их индексу в исходной таблице.
4. В таблицу выводятся **все** строки групп (и первая, и дубли — без удаления/пометок).
5. Группы визуально разделяются жёлтой строкой «Группа: дата „…“ + сумма „…“ — N строк».
6. В шапке столбцов:
   - `DATE_COLUMN_AO` помечен `★` (красный) — ключ дублей.
   - `SUM_COLUMN_AO` помечен `✓` (синий) — ключ дублей + используется для сравнения.
7. Экспорт: `dubles_ГГГГ-ММ-ДД.xlsx`, лист «Дубли АО».
   Содержимое: шапка + все строки-дубли подряд (без разделителей групп — те только в HTML).

# Логика шага 2 — сравнение с Банком

1. Собираем `Set` нормированных сумм из `aoData[i][SUM_COLUMN_AO]` (i ≥ dataStart).
2. По банку: для каждой строки проверяем, есть ли `normalizeSum(row[SUM_COLUMN_BANK])` в этом Set.
3. В результат идут строки Банка, у которых суммы **нет** в АО.
4. В шапке столбцов `SUM_COLUMN_BANK` помечен `★` (красный) — столбец сравнения.
5. Экспорт: `result_ГГГГ-ММ-ДД.xlsx`, лист «Результат».
   Содержимое: шапка Банка + строки без соответствия.

# Состояние (глобальные переменные внутри IIFE)

```
aoData          — массив массивов (из sheet_to_json header:1), обновляется ТОЛЬКО при загрузке файла
bankData        — то же для Банка
dupsState       — { headers, colsCount, groups: [{date, sum, rows:[{idx, row}]}] } или null
resultHeaders   — шапка Банка
resultRows      — строки результата сравнения
```

# Что **не** меняет программа

- **Не модифицирует загруженные файлы** и не модифицирует `aoData`/`bankData` после загрузки.
  Это было изменено: раньше дубли удалялись из `aoData`, сейчас — нет.
- **Не сохраняет состояние** между перезагрузками страницы.

# Зависимости

- `<script src="https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js">` — SheetJS, читает/пишет Excel.
  Если CDN недоступен — выдаст понятную ошибку «Библиотека XLSX не загружена (проверьте интернет)» в плашке файла.
- Никаких npm/node-сборок, всё работает в браузере.

# Известные ограничения

- Всегда берётся **первый лист** книги (`wb.SheetNames[0]`).
- Файлы .csv/.ods/прочие не поддерживаются.
- SheetJS 0.18.5 не читает зашифрованные/повреждённые .xlsx.
- Сравнение сумм — числовое, до копеек. Если в одном файле сумма = `1 234,56`,
  а в другом `1234.560` — оба нормализуются в `1234.56` и совпадут.
- Дата должна быть в одном из поддерживаемых форматов; иначе строка выпадает
  из поиска дублей (но не вызывает ошибку).
